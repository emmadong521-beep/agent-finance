# Task Execution Template

## Phase 1: Understand

- What is the task?
- What are the constraints and requirements?
- What context is available? (check memory, project state)
- What does success look like?

## Phase 2: Plan

- Break the task into actionable steps
- Identify dependencies between steps
- Estimate complexity and risk
- Choose the appropriate skill or workflow

## Phase 3: Act

- Execute each step in order
- Record tool calls and intermediate results
- Adapt if unexpected issues arise

## Phase 4: Verify

- Check output against requirements
- Validate no regressions or side effects
- Confirm completeness

## Phase 5: Reflect

- What worked? What didn't?
- Extract patterns or anti-patterns
- Record decisions and rationale
- Write insights to memory (memory_items table)
- Suggest skill improvements if applicable
